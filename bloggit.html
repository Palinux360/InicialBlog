Guía de Comandos Git

Configuración Inicial y Crear repositorio local

git init

Inicializa un nuevo repositorio de Git en el directorio actual.

git config

Muestra o cambia la configuración de Git.

git config --list

Muestra todas las configuraciones actuales de Git.

git config --global user.name “Nombre Apellido”

git config --global user.email “tu@email.com”

Comandos Git

git log

Muestra los cambios que han ocurrido en el repositorio.

git reset

Permite volver a una versión anterior.

  • git reset <commit>: Resetea al commit especificado.
  • git reset --hard: Vuelve al estado anterior.
  • git reset --soft: Vuelve a la versión anterior mantiene en staging.

git status

Estado actual del repositorio.

git add .

Agrega al staging todos los archivos que están en la carpeta.

git commit -m "mensaje"

Guarda los cambios en el repositorio.

git commit -am

Realiza un commit de todos los cambios en los archivos rastreados. No incluye nuevos archivos no rastreados con git add.

git show

Muestra detalles del commit apuntado por el HEAD.

git diff

Compara versiones, muestra los cambios de la versión en staging vs actual en PC.

git log --stat

Muestra los cambios específicos y en cuáles archivos.

git checkout

Permite ver cómo era el archivo "antes".

  • git checkout <commit>: Cambia al commit especificado.
  • git checkout master nombre.ext: Cambia a la última versión de nombre.

Ramas (Branches de Git)

git branch

Muestra las ramas actuales y destaca la rama actual.

git branch Name

Crea una nueva rama con el nombre “Name”.

git checkout nombre_rama

Cambia a la rama nombre_rama.

git checkout master

Cambia a la rama master.

Fusión de Ramas (Merge)

git merge

Un merge integra cambios de una rama en otra. El merge se realiza en la rama "destino". Ejemplo: si tienes una rama feature y quieres integrar esos cambios en master, primero debes ir a master (git checkout master) y luego realizar el merge (git merge feature).

Comandos Git en Repositorios Remotos

git clone <url>

Trae los archivos del repositorio remoto al directorio de trabajo (copia el master).

git push

Envía el cambio desde el repositorio local al remoto.

git fetch

Trae los cambios del repositorio remoto al repositorio local; no actualiza los archivos de trabajo.

git merge

Fusiona la rama especificada con la rama actual.

git pull

Combina git fetch y git merge. Primero, copia los cambios del repositorio remoto al local y luego fusiona esos cambios en la rama actual.
